Interactive Analysis of Formula 1 Qualifying Data

Norah Kuduk, Matt Higham (Advisor)


Department of Statistics, St. Lawrence University

What is Formula 1 (F1)?

  • F1 is an international motorsport series governed by the Fédération Internationale de l’Automobile (FIA), involving 20 drivers across 10 teams that compete in 18-24 race weekends across a season.

  • Each weekend features practice, qualifying, and race sessions, with qualifying laps representing peak performance used to decide the starting order for the race.

  • A car’s fastest lap from a qualifying session highlights driver skill, car setup efficiency, and team strategy under peak performance conditions.

Objectives

  1. Compare driver performance across races/seasons, thinking about historically important or interesting sessions.

  2. Provide an interactive, visual platform to compare qualifying laps for users.

Methods & Tools

  • Using f1dataR package (R version of fastf1 for Python, Ergast API, etc.).

  • this is an extra bullet point to tray and fix spacing

  • Creation of a local R package (f1animateR) that uses some functions from f1dataR, as well as helper functions when using gganimate.

  • Using Shiny to create the app, hosted on the St. Lawrence University server.

Results

  • As a case study, we are looking at laps from the São Paulo Grand Prix, both across seasons and across different drivers.

Variable Details

Variable Name Description
Driver A 3-letter unique abbriviation of a drivers last name.
Season The year of the session.
Lap Time The time (in seconds) of the lap.
Compound The name of the tire compund the driver is using. Different compounds have different grip levels.

Conclusion

Data is a very large part of Formula 1. Visual analytics offer deeper insight into qualifying performance for the average viewer. Interactive tools like Shiny make F1 data more accessible and fun.
